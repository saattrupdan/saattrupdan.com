# AGENTS.md

Notes for agents working in this repo. The repo is Dan Saattrup Smart's personal
website, deployed to Vercel at https://www.saattrupdan.com.

## Stack

- **Frontend**: Vue 3 + Vue Router 4 + Pinia 2, built with Vite 6, TypeScript.
- **Markdown blog posts**: rendered via `vite-plugin-md` (posts are `.md` files
  treated as Vue SFCs — they can contain `<router-link>` etc.).
- **Math**: MathJax 3 loaded from CDN in `index.html`. Inline `$...$` or
  `\(...\)`; display `$$...$$` or `\[...\]` (configured in `index.html`).
- **Highlighting**: `highlight.js` core + per-language registrations in
  `src/frontend/views/PostView.vue` (only the languages used by posts are
  bundled; add new ones there if you introduce a new code-fence language).
- **Backend**: tiny Python (>=3.11) helpers under `src/backend/`, run via `uv`.
  Only used at build / deploy time; nothing is served from Python.
- **Deploy**: Vercel SPA (all paths rewritten to `index.html` — see
  `vercel.json`). Sitemap is generated at build time by `vite-plugin-sitemap`.
- **Feed**: `public/atom.xml`, regenerated by `src/backend/generate_rss.py`.
- **Analytics**: Piwik Pro snippet in `index.html`.

## Layout

```
src/
  frontend/
    main.ts                # app bootstrap; registers Pinia, router, v-click-outside
    App.vue                # header + <router-view> + footer; menus hidden via meta.showMenus
    router/
      index.ts, routes.ts  # routes; "/" = About, "/posts" = Blog, "/posts/:id" = Post, etc.
    views/                 # one Vue component per route
    components/            # NavMenu, TheHeader, TheFooter, PostSnippet, PaperAbstract, TalkBox, ...
    posts/                 # blog posts as YYYY-MM-DD-slug.md with YAML frontmatter
      postNames.ts         # derives slugs from this dir via import.meta.glob
    papers.yaml            # research papers data
    projects.yaml          # projects data
    talks.yaml             # talks/podcasts/webinars data
    about.md               # About page content
    assets/                # main.css, css-variables.yaml, img/
    directives/            # v-click-outside
    stores/                # Pinia (currently just hamburger menu state)
  backend/
    generate_rss.py        # writes public/atom.xml (pure — makefile handles git)
public/                    # static assets: atom.xml, cv.pdf, favicon.ico, diagrams/, notes/, talks/
index.html                 # MathJax + Piwik + #app mount
vite.config.ts             # Vue + Markdown + YAML + Sitemap plugins; "@" → src/frontend
vercel.json                # SPA rewrite + atom.xml caching headers
makefile                   # convenience targets (install, run, deploy, generate_*)
pyproject.toml / uv.lock   # Python deps (uv-managed)
```

Path alias: `@/` → `src/frontend/`. TypeScript config restricts compilation to
`src/frontend/**`.

## Conventions

### Blog posts
- File name: `src/frontend/posts/YYYY-MM-DD-kebab-slug.md`. The date prefix is
  load-bearing — it drives ordering (`generate_post_names.py`) and Atom feed
  dates (`generate_rss.py`).
- Frontmatter must start with `---` on line 1 and contain at least `title`. Use
  `meta:` (preferred) or `description:` for the feed summary, and `tags:` for
  comma-separated tags.
- Post URLs are `/posts/<slug>` (no `.md`). Internal links to other posts use
  `<router-link to="/posts/<slug>">…</router-link>`, not `[text](url)`, so SPA
  navigation works.
- Post slugs are discovered automatically from the filesystem:
  `vite.config.ts` reads `src/frontend/posts/` with `fs.readdirSync`, and the
  app uses `import.meta.glob` in `postNames.ts`. No generator step; no manual
  list to keep in sync.

### Data files (YAML)
- `papers.yaml`, `projects.yaml`, `talks.yaml` are imported directly via
  `vite-plugin-yaml2` and rendered by the corresponding view. Schema is whatever
  the relevant component expects — check `PaperAbstract.vue`, `ProjectBox.vue`,
  `TalkBox.vue` before adding fields.
- `talks.yaml` items have a `kind` of `talk`, `podcast` or `webinar`, and use
  either `url`, `embed` (iframe src), with optional `embedRatio` / `embedHeight`.

### Routing
- All "real" pages have `meta.showMenus: true`. Standalone embed pages
  (feedback forms, demos) set `showMenus: false` so header/footer are hidden.
- Aliases are used for friendly URLs (e.g. `/` ↔ `/aboutme`, `/posts` ↔
  `/index`, `/talks` ↔ `/podcasts`, `/posts/:id` ↔ `/:id`). Don't add the same
  alias to multiple routes — only the first will resolve.
- Adding a route: edit `src/frontend/router/routes.ts`. The sitemap picks it up
  automatically from `routes.map(r => r.path)` in `vite.config.ts`.

### Style & tooling
- `npm run lint` runs prettier + eslint + `vue-tsc --noEmit` on staged files
  via `lint-staged`. The pre-commit hook is wired up by `simple-git-hooks`
  on `npm install` (`prepare` script).
- Python helpers follow ruff defaults (see `.ruff_cache/` dirs). Keep them
  small and stdlib-ish; the only declared dep is `pyyaml`.

## Things to watch out for

- **`generate_rss.py` writes `public/atom.xml` and exits** — committing and
  pushing the file is the `make deploy` target's job.
- **SPA + trailing slash**: `vercel.json` sets `trailingSlash: false` and
  rewrites everything to `index.html`. Don't add server-side routes — there's
  no server.
- **MathJax 3** config lives inline in `index.html`. Inline math: `$...$` or
  `\(...\)`; display: `$$...$$` or `\[...\]`. Re-typesetting after route
  navigation is triggered in `PostView.vue` via `MathJax.typesetPromise()`.
- **Dates in post filenames are `YYYY-MM-DD`-prefixed** — the Atom generator
  parses them. A malformed prefix will crash the feed build. Sort order
  across the site is plain string-descending on the filename stem, which is
  equivalent to date-descending because of the prefix.
- The `.venv/`, `node_modules/`, `dist/`, `.vercel/` directories are local
  artifacts. `dist/` is the Vite build output.

## Common commands

```
make install      # npm install (also wires up simple-git-hooks)
make run          # npm run dev
npm run build     # vue-tsc + vite build
make deploy       # regenerate RSS, commit + push if changed, vercel --prod
make generate_rss # refresh public/atom.xml only (pure — no git ops)
```
